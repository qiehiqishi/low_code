# 低代码平台实现原理

## 项目概述

本项目是一个基于 Vue 3 + TypeScript + Vite 的低代码平台，实现了完整的可视化页面编辑功能，包括组件拖拽、属性配置、实时预览和代码生成。

## 技术栈

### 核心技术
- **Vue 3**：使用 Composition API 和响应式系统
- **TypeScript**：提供类型安全和更好的开发体验
- **Vite**：快速的开发服务器和构建工具
- **Pinia**：Vue 3 官方推荐的状态管理库

### UI 和工具
- **Element Plus**：UI 组件库，提供丰富的表单和交互组件
- **Tailwind CSS**：实用优先的 CSS 框架
- **vue-draggable-plus**：拖拽功能实现库

### 开发工具
- **unplugin-auto-import**：自动导入 API
- **unplugin-vue-components**：组件自动导入
- **vue-tsc**：Vue TypeScript 类型检查

## 核心实现原理

### 1. 数据结构设计

#### 1.1 ComponentData（组件数据结构）

```typescript
interface ComponentData {
  id: string                    // 组件唯一标识
  componentName: string         // 组件名称（如 'el-button', 'div'）
  label?: string                // 显示名称
  props: ComponentProps         // 组件属性对象
  styles: ComponentStyles       // 组件样式对象
  children?: ComponentData[]    // 子组件数组（递归）
  parentId?: string             // 父组件ID
  isContainer?: boolean         // 是否为容器组件
  isLocked?: boolean            // 是否锁定
  isVisible?: boolean           // 是否可见
}
```

**设计理念**：
- **树形结构**：使用 children 数组实现递归嵌套
- **属性分离**：props 和 styles 分开管理，便于独立编辑
- **唯一标识**：每个组件有唯一 ID，便于查找和操作
- **容器标识**：isContainer 字段标识容器组件，支持嵌套

#### 1.2 MaterialComponent（物料定义）

```typescript
interface MaterialComponent {
  type: string                  // 组件类型
  label: string                 // 显示名称
  icon?: string                 // 图标
  category: string              // 分类
  defaultProps: ComponentProps  // 默认属性
  defaultStyles?: ComponentStyles // 默认样式
  schema?: PropSchema[]         // 属性配置 schema
  styleSchema?: PropSchema[]    // 样式配置 schema
  isContainer?: boolean         // 是否为容器
}
```

**设计理念**：
- **Schema 驱动**：使用 schema 定义属性和样式配置
- **默认值**：提供合理的默认值，减少用户配置
- **分类管理**：按类别组织物料，便于查找

#### 1.3 PropSchema（属性配置 Schema）

```typescript
interface PropSchema {
  name: string                  // 属性名
  label: string                 // 显示名称

  options?: { label: string; value: any }[] // 选项

```

**设计理念**：
- **类型驱动**：根据类型自动生成表单
- **选项支持**：select 类型提供选项列表
- **验证支持**：required 字段支持必填验证

#### 6.2 属性转换

**实现**：
```typescript
function generateAttributes(component: ComponentData): string {
  let attributes = ''
  
  for (const [key, value] of Object.entries(component.props)) {
    // 布尔值属性
    if (typeof value === 'boolean') {
      if (value) attributes += ` ${key}`
    }
    // 字符串和数字属性
    else if (value !== undefined && value !== null && value !== '') {
      attributes += ` ${key}="${value}"`
    }
  }
  
  return attributes
}
```

**关键点**：
- **布尔值处理**：true 仅输出属性名，false 不输出
- **值过滤**：过滤无效值（undefined、null、空字符串）
- **格式化**：输出标准 HTML 属性格式

#### 6.3 样式转换

**实现**：
```typescript
function generateStyles(styles: Record<string, any>): string {
  const styleArray: string[] = []
  
  for (const [key, value] of Object.entries(styles)) {
    if (value !== undefined && value !== null && value !== '' && value !== 'auto') {
      const cssKey = camelToKebab(key) // 驼峰转短横线
      styleArray.push(`${cssKey}: ${value}`)
    }
  }
  
  return styleArray.join('; ')
}
```

**关键点**：
- **命名转换**：驼峰命名转短横线命名（backgroundColor → background-color）
- **值过滤**：过滤无效值和默认值（auto）
- **拼接格式**：使用分号分隔样式

#### 6.4 嵌套处理

**实现**：
```typescript
function generateTemplate(components: ComponentData[], indent: number = 1): string {
  const indentStr = '  '.repeat(indent)
  let result = ''
  
  for (const component of components) {
    const tagStart = generateTagStart(component, indentStr)
    const tagEnd = generateTagEnd(component, indentStr)
    
    if (component.children && component.children.length > 0) {
      const childrenContent = generateTemplate(component.children, indent + 1)
      result += `${tagStart}\n${childrenContent}\n${tagEnd}\n`
    } else {
      result += `${tagStart}${generateTagContent(component)}${tagEnd}\n`
    }
  }
  
  return result.trim()
}
```

**关键点**：
- **递归生成**：递归处理子组件
- **缩进控制**：每层增加 2 个空格缩进
- **子组件处理**：子组件增加缩进层级（indent + 1）

### 7. 历史记录实现

#### 7.1 撤销/重做机制

**实现**：
```typescript
const history = ref<ComponentData[][]>([])
const historyIndex = ref<number>(-1)

const saveHistory = () => {
  // 删除当前索引之后的历史记录
  history.value = history.value.slice(0, historyIndex.value + 1)
  
  // 添加当前状态到历史记录（深拷贝）
  history.value.push(JSON.parse(JSON.stringify(components.value)))
  historyIndex.value = history.value.length - 1
}

const undo = () => {
  if (historyIndex.value > 0) {
    historyIndex.value--
    components.value = JSON.parse(JSON.stringify(history.value[historyIndex.value]))
    selectedComponent.value = null
  }
}

const redo = () => {
  if (historyIndex.value < history.value.length - 1) {
    historyIndex.value++
    components.value = JSON.parse(JSON.stringify(history.value[historyIndex.value]))
    selectedComponent.value = null
  }
}
```

**关键点**：
- **深拷贝**：使用 JSON 序列化实现深拷贝
- **索引管理**：historyIndex 指向当前状态
- **状态恢复**：从历史记录恢复组件树
- **选中清空**：撤销/重做时清空选中状态

### 8. 组件查找实现

#### 8.1 递归查找

**实现**：
```typescript
const findComponentById = (id: string): ComponentData | null => {
  for (const root of components.value) {
    const found = findComponentRecursive(root, id)

const findComponentRecursive = (node: ComponentData, id: string): ComponentData | null => {
  if (node.id === id) return node
  
  if (node.children) {
    for (const child of node.children) {
      const found = findComponentRecursive(child, id)
      if (found) return found
    }
  }
  
  return null
}
```

**关键点**：
- **递归遍历**：递归遍历整个组件树
- **深度优先**：使用深度优先搜索算法
- **性能考虑**：对于大型组件树可能需要优化

## 数据流设计

### 1. 单向数据流

```
用户操作 → Store Actions → State 更新 → 组件渲染
```

**流程**：
1. 用户在画布点击组件
2. 触发 `selectComponent` action
3. 更新 `selectedComponent` state
4. 属性面板自动显示选中组件的配置

### 2. 响应式更新

```
State 变化 → Computed 计算 → 组件自动更新
```

**示例**：
```typescript
// State
const components = ref<ComponentData[]>([])

// Computed
const componentCount = computed(() => {
  const result: ComponentData[] = []
  const flatten = (node: ComponentData) => {
    result.push(node)
    if (node.children) {
      node.children.forEach(child => flatten(child))
    }
  }
  components.value.forEach(root => flatten(root))
  return result.length
})

// 组件自动更新
<span>组件数: {{ editorStore.componentCount }}</span>
```

### 3. 事件传递

```
子组件事件 → 父组件监听 → 顶层处理 → Store 更新
```

**示例**：
```vue
<!-- 子组件 -->
<ComponentRenderer
  @select="(id) => emit('select', id)"
  @hover="(id) => emit('hover', id)"
/>

<!-- 父组件 -->
<ComponentRenderer
  @select="handleChildSelect"
  @hover="handleChildHover"
/>

<!-- 顶层 -->
const handleSelect = (id: string) => {
  const component = editorStore.findComponentById(id)
  editorStore.selectComponent(component)
}
```
